# AI Leverage Case Sheet

Use this sheet to make AI claims concrete, credible, and enterprise-safe. Each case provides a reusable template for discussing AI-augmented engineering in interviews.

## Case A — AI-Assisted Design and Implementation Flow

- **Problem:** Long cycle time from problem framing to implementation for complex backend/platform work.
- **AI leverage:** Used coding agents for design iteration, code scaffolding, and option exploration.
- **Human judgment points:** Architecture boundaries, risk tradeoffs, and final design decisions.
- **Quality controls:** Mandatory review checklist, test coverage threshold, and production-readiness gate.
- **Outcome placeholders:** Cycle time from design to first production candidate, review turnaround time.

## Case B — AI-Augmented Refactoring and Reliability Improvements

- **Problem:** Refactoring and reliability work often under-prioritized due to perceived delivery cost.
- **AI leverage:** Accelerated low-risk refactors, test updates, and runbook/documentation drafts.
- **Human judgment points:** Deciding what to refactor, phased rollout strategy, incident risk management.
- **Quality controls:** Canary rollout, observability checks, rollback readiness.
- **Outcome placeholders:** Reliability trend, defect trend, and time-to-ship improvements.

## Case C — Team-Level Enablement Pattern

- **Problem:** Uneven productivity and quality outcomes across engineers using AI tools inconsistently.
- **AI leverage:** Established repeatable usage patterns and prompts for design, coding, and review.
- **Human judgment points:** Policy guardrails, secure coding standards, architecture review approvals.
- **Quality controls:** Code review rubric, security scan requirements, and explicit ownership boundaries.
- **Outcome placeholders:** Team throughput change, review quality consistency, onboarding velocity.

## Guardrail Narrative (Use in Interviews)

"AI accelerates discovery and implementation, but quality comes from disciplined engineering: architecture reviews, testing rigor, observability, security checks, and clear ownership. The value is not one-shot generation — it's building a repeatable system for higher-quality delivery."

## Metrics To Fill Before Interviews

- Baseline vs post-AI cycle time
- Defect escape rate before/after
- Rollback rate before/after
- Review latency and merge latency trends
- Team adoption rate and consistency metrics
- Concrete examples of AI-augmented deliverables with quality outcomes
