# Content Review

## Purpose

Provide a structured editorial review with clear fixes and stronger rewritten copy. Automatically adapts depth and tone to the content type — professional writing, technical narratives, architecture docs, or thought leadership.

## Default Mode

Structured editorial output:
1. Strengths
2. Issues
3. Rewrite
4. Rationale

For technical content, also include:
- Inferred audience and style choice
- Accuracy or ambiguity flags
- Optional alternate version for a second audience

## Use When

- Editing professional writing, job-search docs, and narrative drafts.
- Turning rough text into concise, audience-aware communication.
- Editing architecture write-ups or technical narratives.
- Preparing engineering-facing or executive-facing technical content.
- Translating deep technical ideas for mixed audiences.
- Improving message clarity and flow without changing intent.

## Workflow

1. Identify audience and purpose. For technical content, infer audience level (executive, cross-functional, engineering).
2. Diagnose strongest and weakest sections.
3. Evaluate clarity, precision, and logical flow. For technical content, also check jargon density and accuracy.
4. Rewrite with tighter structure and stronger verbs.
5. Explain why the rewrite is better.

## Required Output

- Top strengths (short)
- Key issues (prioritized)
- Full rewrite or targeted rewrite
- Rationale for major edits
- For technical content: inferred audience, accuracy flags, and optional alternate-audience version
